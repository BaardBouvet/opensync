# Plans

Design and implementation plans. Each file here starts as a backlog item. Once work begins it moves
to in-progress, and when complete the plan is converted into a spec and/or code and removed from
this directory.

## Statuses

| Status | Meaning |
|--------|---------|
| `backlog` | Identified, not yet started |
| `in-progress` | Actively being implemented |
| `done` | Implemented — plan pending conversion to spec/code |
